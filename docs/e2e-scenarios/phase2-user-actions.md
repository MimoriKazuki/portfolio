# Phase 2：ユーザーアクションテスト（CRUD・フォーム・画面遷移）

## 目的

各画面で **ユーザーが実際に行う操作** を検証する。
- フォーム入力・バリデーション
- 登録・更新・削除（CRUD）
- 画面遷移
- 検索・絞り込み・フィルタ

業務一気通貫のフロー（複数画面跨ぎ）は Phase 3 で扱う。

## シナリオ作成方針

- 画面ごとに「主要操作」をシナリオ化
- ID 体系：`SC-UAT-{連番3桁}`
- 1画面につき 5〜20件のシナリオが目安
