# Phase 3：E2Eフローテスト（業務一気通貫）

## 目的

複数画面を跨ぐ **業務フロー全体** を検証する。
Phase 3 WBS（docs/wbs/phase3.md）の Step 4 タスク P3-E2E-01〜06 に対応する。

## シナリオ作成方針

- 業務フローのフェーズ単位でシナリオ化
- ID 体系：`SC-E2E-{連番3桁}`
- 各シナリオは「実際の業務手順を最初から最後まで」再現
- 本番データ保護：テストデータは `[E2E_TEST]` プレフィックスで作成・完了後クリーンアップ

## シナリオ一覧

### SC-E2E-001: LP → ログイン → コース一覧（P3-E2E-01）

- WBS ID: P3-E2E-01
- spec: `e2e/scenarios/lp-to-courses.spec.ts`
- 対象業務: 業務フロー「2. LP流入」→「3. 会員登録/ログイン」→「4. コンテンツ閲覧」
- ステップ:
  1. `/e-learning`（B001 LP）に未ログインで訪問
  2. ヒーローセクション・コース紹介が表示されることを確認
  3. 「コースを見る」CTA をクリック → `/auth/login?returnTo=/e-learning/home` に遷移
  4. Google OAuth でログイン
  5. `/e-learning/home`（B002）に遷移することを確認
  6. コース一覧・単体動画タブが表示されることを確認
  7. コースカードをクリック → B004 コース詳細に遷移することを確認
- 検証ポイント:
  - returnTo クエリが正しく処理される
  - ログイン後に B002 会員ホームが表示される
  - コース一覧が正常に描画される
- ステータス: 📋 未着手

### SC-E2E-002: 購入フロー（P3-E2E-02）

- WBS ID: P3-E2E-02
- spec: `e2e/scenarios/purchase-flow.spec.ts`
- 対象業務: 業務フロー「5. 購入決済」→「6. Webhook受信」→「7. コンテンツ視聴」
- 前提: ログイン済み・有料コースが1件存在する（テスト用）
- ステップ:
  1. B004 コース詳細（有料・未購入）へアクセス
  2. 「購入する」CTA をクリック → B008 購入確認モーダルが開く
  3. モーダル内容（コース名・価格）を確認
  4. 購入確認ボタンをクリック → Stripe Checkout にリダイレクト
  5. （Stripe Test Mode）テスト決済情報を入力して完了
  6. B009 購入完了画面 `/e-learning/checkout/complete?session_id=...` に遷移
  7. 購入対象コースへの導線が表示されることを確認
  8. 導線をクリック → B005 視聴画面で動画が再生開始されることを確認
- 検証ポイント:
  - 購入前は視聴画面に直アクセスできない
  - Webhook 反映後に視聴権限が付与される
  - B009 ポーリングが正常動作する
- ステータス: 📋 未着手

### SC-E2E-003: 視聴・進捗（P3-E2E-03）

- WBS ID: P3-E2E-03
- spec: `e2e/scenarios/watch-progress.spec.ts`
- 対象業務: 業務フロー「7. コンテンツ視聴」→ 進捗管理
- 前提: ログイン済み・視聴権限のあるコースが存在する
- ステップ:
  1. B005 コース視聴画面に直アクセス
  2. 動画プレイヤーが表示されることを確認
  3. 動画を最後まで視聴（またはテスト用完了操作）
  4. 視聴済みアイコンが表示されることを確認
  5. マイラーニング（B012b）`/e-learning/lp/mypage/learning` で購入済みタブを確認（B013 視聴履歴は削除済み・代替なし）
  6. 「次レッスン」ボタンをクリック → 次の動画が再生される
  7. 章内全動画を完了 → コース進捗率が100%になることを確認
- 検証ポイント:
  - progress レコードが正しく作成される
  - 進捗率の計算が正確
  - 次レッスン遷移が連続して機能する
- ステータス: 📋 未着手

### SC-E2E-004: ブックマーク追加・一覧・解除（P3-E2E-04）

- WBS ID: P3-E2E-04
- spec: `e2e/scenarios/bookmark.spec.ts`
- 対象業務: 業務フロー「8. ブックマーク登録」
- 前提: ログイン済み
- ステップ:
  1. B003 コース一覧でコースカードのブックマークアイコンをクリック
  2. アイコンがアクティブ状態になることを確認
  3. マイラーニング（B012b）`/e-learning/lp/mypage/learning?tab=bookmarked` へ遷移
  4. ブックマークしたコースが一覧に表示されることを確認
  5. B004 コース詳細画面でもブックマークがアクティブなことを確認
  6. B004 コース詳細画面でブックマーク解除 → マイラーニング「ブックマーク済み」タブから消えることを確認
  7. B003 コース一覧で該当コースのアイコンが非アクティブになることを確認
  （クリーンアップ：テスト用ブックマーク削除）
- 検証ポイント:
  - ブックマーク追加・解除が複数画面で即時反映される
  - e_learning_bookmarks テーブルへの記録が正確
- ステータス: 📋 未着手

### SC-E2E-005: 退会 → 再登録（履歴引継確認）（P3-E2E-05）

- WBS ID: P3-E2E-05
- spec: `e2e/scenarios/withdraw-relogin.spec.ts`
- 対象業務: 業務フロー「退会・アカウント削除」→ 再登録
- 前提: テスト用アカウントでログイン済み（本番アカウントとは別のテスト用）
- ステップ:
  1. B014 マイページ `/e-learning/mypage` にアクセス
  2. 退会ボタンをクリック → 確認ダイアログが表示される
  3. 退会を確定 → セッション破棄・B001 LP にリダイレクトされる
  4. 同一 Google アカウントで再ログイン
  5. B002 会員ホームに遷移することを確認
  （クリーンアップ：テスト用アカウント状態を元に戻す）
- 検証ポイント:
  - 退会後にセッションが破棄される
  - 再ログイン時に正常に会員ホームへ遷移できる
- ステータス: 📋 未着手

### SC-E2E-006: 管理画面 コース作成→章追加→動画追加→公開（P3-E2E-06）

- WBS ID: P3-E2E-06
- spec: `e2e/scenarios/admin-course-create.spec.ts`
- 対象業務: 業務フロー「1. 商品（コース）の整備」
- 前提: 管理者アカウントでログイン済み
- ステップ:
  1. C005 コース一覧 `/admin/e-learning/courses` にアクセス
  2. 「新規作成」ボタンをクリック → C006 コース新規作成画面へ遷移
  3. 基本情報タブ：コース名「[E2E_TEST] テストコース」・カテゴリ・価格を入力
  4. カリキュラムタブ → 「章を追加」→ 章名「第1章 テスト」を入力
  5. 第1章内に「動画を追加」→ 動画情報を入力
  6. 公開フラグをオンにして保存
  7. C005 一覧に作成したコースが表示されることを確認
  8. B003 コース一覧（会員側）に公開コースが表示されることを確認
  （クリーンアップ：C007 編集画面から論理削除）
- 検証ポイント:
  - コース・章・動画の階層構造が正しく保存される
  - 公開フラグ変更が会員側画面に即時反映される
  - 論理削除後に会員側から非表示になる
- ステータス: 📋 未着手
