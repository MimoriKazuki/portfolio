# Phase 5：パフォーマンステスト

## 目的

各画面・操作の **パフォーマンス（応答時間・レンダリング時間）** を計測する。
本番想定のデータ量で、許容範囲内に収まっているかを検証。

## シナリオ作成方針

- 大量データ表示・複雑な集計・購入フロー等を中心にシナリオ化
- ID 体系：`SC-PRF-{連番3桁}`
- 各シナリオに「目標値」と「合格ライン」を明記

## 計測指標

- **TTFB**（Time To First Byte）：1秒以内
- **FCP**（First Contentful Paint）：1.5秒以内
- **LCP**（Largest Contentful Paint）：2.5秒以内
- **TTI**（Time To Interactive）：3秒以内
- **API 応答時間**：500ms 以内（標準）、3秒以内（集計系）
