# デザイン方針（design-direction.md）

このプロジェクトのデザインの「源流」を確定するドキュメント。
カラー・タイポグラフィ・スペーシング等の具体的トークンは `tokens/` 配下、レイアウト規約は `layout/` 配下、NGパターンは `ng-patterns.md` で定義する。
ここでは **「誰のために・何のために・どんな印象で」** を業務的に決定する。

## ★最重要前提：本ドキュメントは「既存実装からの抽出」である

本プロジェクトは **本番運用中**（https://www.landbridge.ai/）であり、本ドキュメントは **既存稼働中サイトから抽出した「らしさ」を言語化した結果** である。**新たな方向性を打ち出すものではない。**

ディレクター方針 **「既存のあしらいに合わせる」** に従い、印象キーワード・デザインコンセプト・方針マトリクスのいずれも、既存実装（`tailwind.config.ts` / `app/globals.css` / `app/components/`）から抽出し言語化したものを正とする。

Phase 1 で追加する LP・コース系・管理画面の新規 UI は、いずれも **既存実装のあしらいを必ず踏襲する**（新規創出ではなく、既存パターンの再利用と整理が原則）。

### 抽出元の既存実装（本ドキュメントの根拠）

| 抽出領域 | 既存実装 |
|---|---|
| LP のヒーロー・スクロール演出・数値強調・カード | `app/components/AITrainingLP.tsx` / `ServiceTrainingLP.tsx` |
| コース／単体動画カードのあしらい | `app/e-learning/ELearningCard.tsx` / `ELearningTopClient.tsx` |
| ヘッダー・フッター・サイドバー・レイアウト | `app/components/Header.tsx` / `MobileHeader.tsx` / `Sidebar.tsx` / `Footer.tsx` / `MainLayout.tsx` |
| 管理画面の DataTable・フォーム | `app/admin/*` 配下 |
| UI プリミティブ（ボタン・モーダル・フォーム） | `app/components/ui/`（shadcn ベース） |
| カラー・角丸・フォント等のトークン源 | `app/globals.css`（CSS 変数） / `tailwind.config.ts` |

## DG-A：要件・目的整理＋リサーチ

### ターゲットユーザー像

#### 主要利用者（会員ページ）
- **役割**：Eラーニング受講者（個人会員・法人会員のエンドユーザー）
- **属性**：AI活用に関心のあるビジネスパーソン（ノンエンジニア寄り含む）・経営者・人事担当者
- **リテラシー**：中〜高（PC・スマホでの動画視聴・会員登録に抵抗がない）
- **利用環境**：オフィス／自宅／移動中（マルチデバイス）
- **利用デバイス**：PC（主）+ モバイル（副）。視聴はモバイルでも発生する

#### 副次利用者（管理者画面）
- **役割**：弊社運営担当（コンテンツ登録・受講者管理・売上確認）
- **属性**：社内オペレーター・経営層
- **リテラシー**：高（業務 SaaS の操作に慣れている）
- **利用環境**：オフィス常駐
- **利用デバイス**：PC 主（モバイルは閲覧のみ・複雑な編集 UI は PC 前提）

#### 副次利用者（LP / 未ログイン来訪者）
- **役割**：購入検討者・無料コンテンツの試聴者
- **リテラシー**：低〜高（幅広い）
- **利用デバイス**：モバイル多め（広告流入想定）

### 主要アクション

#### 会員ページ
1. コース／単体動画を探す（一覧・検索）
2. 動画を視聴する・章を進める
3. ブックマーク／資料 PDF を取得する
4. プラン／コースを購入する

#### 管理者画面
1. コンテンツ・コース・カテゴリの登録／編集
2. 受講者の権限管理（個人・法人）
3. 売上・視聴データの確認

#### LP
1. サービス内容を理解する
2. 会員登録・購入へ進む

### 与えたい印象キーワード（4個・★ディレクター承認済 2026-05-12）

**以下は既存 LP（`AITrainingLP.tsx` / `ServiceTrainingLP.tsx`）・既存カード（`ELearningCard.tsx`）・既存管理画面（`/admin/*`）から抽出されたキーワード**である。新規方向性ではなく、既に実装されているトーンの言語化。

| キーワード | 理由・根拠（既存実装での現れ方） |
|---|---|
| プロフェッショナル | AI駆動研究所のブランド（経営層・人事担当も顧客）として、既存 LP の硬すぎない実直な実装で表現済み |
| スマート（知的・洗練） | AI / テック領域として時代性のある洗練感。既存 `AITrainingLP.tsx` の数値強調・グラデ・余白で表現済み |
| 信頼感 | 業務利用・法人研修も想定。既存 `--primary`（青系 HSL `221.2 83.2%`）+ ニュートラルな無彩色ベースで表現済み |
| 集中・ノイズなし | 既存 `ELearningCard.tsx` の枠線ベース・控えめシャドウ・静謐な白基調で表現済み |

※ 印象キーワードは既存実装からの抽出結果として承認（2026-05-12 / Kosuke）。

### 参考サイト（既存実装そのものを最大の参考とする）

| URL／参照 | 参考にしたいポイント | 該当ページ・要素 |
|---|---|---|
| 既存サイト https://www.landbridge.ai/ | カラー使い・ヘッダー構造・LP のスクロール演出 | TOP / `/e-learning` / `/services` 系 LP |
| 既存 `AITrainingLP.tsx` / `ServiceTrainingLP.tsx` | ヒーロー・グラデーション背景・数値強調・アイコン併記・スクロール連動 | LP セクション全般 |
| 既存 `ELearningCard.tsx` / `ELearningTopClient.tsx` | カードのあしらい（`aspect-video` + バッジ + 再生オーバーレイ）・無料／有料の色分け | コース／単体動画カード |
| 既存 `/admin/*` 配下 | DataTable のレイアウト・フォーム配置 | 管理画面全般 |

外部参考サイトは、新規にデザインを起こす場合のみ plan-lead 経由で `@design-direction-helper` を起動して別途調査する。Phase 1 スコープでは既存パターンの踏襲を最優先とする。

### 競合・類似サービスのトレンド

- **領域**：日本のオンライン学習プラットフォーム（Udemy / Schoo / Aidemy 等）+ AI/DX 研修系 SaaS
- **主流のスタイル**：
  - 白＋濃色見出し＋プライマリ青／青緑系のアクセント
  - カードレイアウト中心（動画一覧）
  - LP は数値強調・受講者の声・大きめヒーロー
- **業界の特徴**：派手なグラデは避け、信頼性訴求が中心。動画サムネが視覚の主役

### NG項目（やってほしくないこと）★最重要

| NG項目 | 理由 |
|---|---|
| 派手な原色多用・ネオン系・極端な高彩度 | AI研修サービスの信頼感を損なう |
| パステル系・かわいい系トーン | ターゲット（経営層・人事）に合致しない |
| 過剰なアニメーション（バウンス・連続トランジション） | 視聴・学習体験を妨げる |
| 既存実装と大きく異なる独自スタイルの新規導入 | ディレクター方針「既存のあしらいに合わせる」に反する |
| ライト／ダーク両前提のクラス（`dark:` の常用） | Phase 1 ではダーク対応をしない方針（ライト前提） |
| `bg-[#xxxxxx]` の arbitrary value 多用 | トークン化を崩す（既存も基本 Tailwind スケールで完結） |
| 色のみで状態を伝える（無料／有料・視聴可／不可） | アクセシビリティ違反・色覚多様性に非対応 |
| 動画自動再生（音あり） | UX 低下・モバイルでは特に嫌われる |
| ボタンに `font-bold`（既定 weight 300 ルールに反する） | 既存 globals.css の重みルールを破る |

### ブランド素材確認

- **ロゴ**：あり（既存ヘッダー `app/components/Header.tsx` / `MobileHeader.tsx` で表示中）
- **既存カラー**：あり
  - プライマリ青：`hsl(221.2 83.2% 53.3%)`（≒ `#2563EB`）= shadcn `--primary`
  - ブランド独自青：`portfolio.blue` = `#3B82F6` / `portfolio.blue-dark` = `#2563EB` / `portfolio.blue-light` = `#60A5FA`
  - 無彩色ベース：`portfolio.gray-light` = `#F9FAFB` / `portfolio.gray` = `#F3F4F6` / `portfolio.gray-dark` = `#E5E7EB`
- **既存フォント**：あり
  - 欧文：`acumin-pro`（Adobe Fonts）
  - 和文：`Noto Sans JP`（next/font/google）
  - 装飾用：`Caveat`（既存 LP の一部に使用）
- **既存システムからの継承**：あり（**全面的に踏襲**）
  - shadcn 標準 CSS 変数（`--primary` / `--background` / `--card` 等）
  - `app/components/ui/`（shadcn ベース UI プリミティブ）
  - 角丸 `--radius: 0.75rem`（12px）
  - レイアウト：`Header` / `MobileHeader` / `Sidebar` / `Footer` / `MainLayout`

## DG-B：デザイン方針言語化

### デザインコンセプト（★ディレクター承認済 2026-05-12）

> **「現場で使える・ノイズのない・信頼感のある業務 AI 学習体験」**

**既存実装のトーン＆マナーを一文に凝縮したもの。新規創出ではない。**
既存 `AITrainingLP.tsx` の硬派な数値訴求、既存 `ELearningCard.tsx` の枠線・余白による静謐さ、既存 `/admin/*` の業務適性 — これらが現に表現している姿を言語化したコンセプト。

補足：経営層・人事担当・現場担当が同じ画面で迷わない、白基調＋青系プライマリの落ち着いた配色と、視聴・購入・進捗確認の 3 動線を明快に示す情報設計を両立する。

※ 承認時の補足（ディレクター）：**「既存実装のあしらいを必ず踏襲する」前提で承認**。新規方向性提案ではなく、現状言語化として確定。

### ムードボード

| 要素 | 表現・参照先 |
|---|---|
| カラートーン | 白＋ニュートラルグレー + 青系プライマリ（`#2563EB` 近似）。アクセント青は `portfolio.blue-light`〜`portfolio.blue-dark` の範囲。差し色（success / warning / error）は彩度を抑える |
| フォント感 | 欧文 `acumin-pro` + 和文 `Noto Sans JP`。見出しは bold / semibold、本文 regular、**ボタンのみ light（300）**（既存 globals.css ルール） |
| 余白感 | 中〜広め。LP は息のある縦余白（`py-16 sm:py-20 lg:py-24`）、管理画面はやや密（`py-6`〜`py-8`） |
| 装飾感 | 角丸 12px（`--radius: 0.75rem`）基準。シャドウは控えめ（`shadow-md` 標準、hover で `shadow-lg`）。グラデは LP ヒーローのみ |
| インタラクション | hover で軽い浮き上がり（`hover:-translate-y-1 hover:shadow-lg`）。fade-in / scale-in / ripple の既存控えめモーションを継承。過剰なバウンス・連続演出は不採用 |
| 情報密度 | LP：低〜中（訴求重視）／会員一覧：中（カード4列まで）／管理：高（DataTable は密度高） |
| キービジュアル | 既存 LP のスクロール連動・大きめ数値・アイコン併記・グラデ背景を踏襲 |

### 方針マトリクス（用途別の使い分け）

画面性質によってデザイン指針が異なる。新規画面はこの表のいずれかに位置付ける。
**「参考実装」列に挙げた既存ファイルのあしらいを必ず踏襲すること**（新規創出ではなく既存パターンの再利用）。

| 画面性質 | 情報密度 | 配色トーン | コンポーネント傾向 | 該当画面（例） | 参考実装（必須踏襲） |
|---|---|---|---|---|---|
| LP（ランディング） | 低〜中 | 白基調＋プライマリ青、ヒーローに薄グラデ | 大きめヒーロー・カード・数値強調・CTA ボタン目立たせ | `/e-learning` 入口 LP（B001） | `app/components/AITrainingLP.tsx` / `ServiceTrainingLP.tsx` |
| 会員一覧（コース／単体動画） | 中 | 白＋カード（`shadow-md`） | グリッドカード（`xs:2 / lg:3 / wide:4`） | コース一覧・動画一覧（B002／B003） | `app/e-learning/ELearningCard.tsx` / `ELearningTopClient.tsx` |
| 詳細・購入画面 | 中 | 白＋カード＋CTA 強調 | ヒーロー＋カリキュラム＋購入カード（右サイド） | コース詳細・単体動画詳細（B004／B007） | 既存 `app/e-learning/[id]/*` の詳細レイアウト |
| 視聴画面 | 中 | 白＋静謐な背景 | 動画プレーヤー＋サイドにレッスン／関連動画（`xl` 以上で横並び） | コース視聴・単体動画視聴（B005／B007） | 既存 `app/e-learning/[id]/*`（再生埋め込み）＋ `MainLayout.tsx` |
| 設定・マイページ | 低 | ニュートラル | シンプルフォーム＋セクション分け | プロフィール／購入履歴（B011〜B014） | 既存 `app/components/ui/`（form / card） |
| 管理 DataTable | 高 | ニュートラル（白＋区切り線） | テーブル＋フィルタ＋一括操作 | コンテンツ／コース／受講者管理（C\*） | 既存 `app/admin/*` 配下（既存 DataTable） |
| 管理フォーム | 中 | ニュートラル | メインフォーム＋右サイドカード（`lg:col-span-8 / 4`） | コンテンツ登録・コース編集（C002〜C007） | 既存 `app/admin/*` 配下のフォーム画面 |
| 認証 | 低 | 中央寄せ・カード | `max-w-md` カード＋Google ログインボタン | ログイン（A001） | 既存 `app/auth/login` |
| エラー・空状態 | 低 | ニュートラル | アイコン＋メッセージ＋CTA | 404 / 空一覧 | 既存 `app/not-found.tsx` + shadcn `Card` |

### コンセプトに反する設計をしないためのチェックリスト

設計・実装時の自己チェック観点（PR レビューでも使う）：
- [ ] 印象キーワード（プロフェッショナル／スマート／信頼感／集中・ノイズなし）のいずれかに反していないか
- [ ] NG項目に該当する表現を持ち込んでいないか（特に派手色・arbitrary value・自動再生・色のみ依存）
- [ ] 方針マトリクスのどの画面性質に属するか明示できるか／その指針に整合しているか
- [ ] 既存実装（`globals.css` / `tailwind.config.ts` / `app/components/`）のトークン・パターンを優先利用しているか
- [ ] ボタン weight 300 ルール・iOS zoom 抑制 16px 等の既存ルールを守っているか
