# フィードバック管理 インデックス

このプロジェクト（AI駆動研究所 ポートフォリオ／メディアサイト・Eラーニング刷新スコープ）のフィードバック（クライアント要望・内部気づき・バグ報告）を一元管理する。

## サマリー

|---------|---------|------|---------|---------|-------|-------|----------|
| 共通・全体 | [system-wide.md](system-wide.md) | SYS | 0 | 0 | 0 | 0 | 0 |
| 認証関連（A001-A002） | [auth.md](auth.md) | AUTH | 0 | 0 | 0 | 0 | 0 |

※ 画面カテゴリは `docs/frontend/screens.md`（Phase 1 確定）に従う。

## ナンバリング

- 形式：`FB-{画面略称}-{連番3桁}` 例：`FB-SYS-001`、`FB-ELN-042`
- 連番は画面ごとに独立（各ファイル内で 001 から採番）
- **削除禁止**：番号は永久保持。不要になった場合はステータスを「➖ 対応不要」に変更

## ステータス種別

| 絵文字 | ステータス | 用途 |
|--------|-----------|------|
| 📋 | 未着手 | 登録済み、未対応 |
| 🔧 | 対応中 | 作業に着手している |
| ✅ | 完了 | 対応完了（ディレクター 目視確認後に変更） |
| ⏸ | 保留 | 一時的に見送り（後で再開の可能性あり） |
| ➖ | 対応不要 | 仕様通り・再現不可・取り下げ等で対応しない |

## 横断タスクの管理

複数画面にまたがるフィードバックは：
1. **起点となる画面にメイン登録**する（最も影響範囲が広い画面が起点）
2. 関連する他の画面ファイルには **参照エントリ** を作成し、メインへのリンクを記載する
3. メイン側には `関連画面` フィールドで全関連先を列挙する

例：
```
[FB-SYS-002] 全画面共通の検索条件追加
- メイン：system-wide.md
- 関連画面：elearning-public.md / elearning-admin.md ...
```

## 運用フロー

1. ディレクター（Kosuke）がフィードバックを認知（クライアント要望 or 自身の気づき）
2. team-lead が該当画面の md ファイルに登録（優先度・分類を判断）
3. team-lead が dev-mate へ対応指示を出す（FB番号を含めた指示）
4. dev-mate が実装・コミット（コミットメッセージに FB番号を含める）
5. review-mate / unittest-mate が検証
6. team-lead がステータスを 🔧対応中 から ✅完了 に更新（ディレクター 目視確認後）
7. INDEX のサマリーテーブルを都度更新

## 各画面別ファイルのフォーマット

各画面別ファイル（system-wide.md / auth.md / elearning-public.md / elearning-admin.md）は同一フォーマットで管理する。
