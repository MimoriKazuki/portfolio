## B. 弊社メンバーステータスの判定方法
1. 「弊社メンバー」は何を見て判定しますか？ 案：(a) auth.users.email がドメイン @landbridge.co.jp、(b) e_learning_users に bool フラグ is_internal、(c) 管理者テーブル所属、(d) その他
2. 既存購入者 6 名は「弊社メンバー扱い」になりますが、社外メールの方も含まれますか？ 含まれる場合、(a) のドメイン判定では拾えないため判定ロジックが変わります
3. 弊社メンバーは「全コンテンツ視聴可・永続」とのことですが、これはサブスク／コース／単品すべてを横断して視聴可能、という理解で合っていますか？
4. 弊社メンバーに有効期限はありますか？（退職時の扱い・無期限か）
5. 弊社メンバーの追加・削除は誰が・どの画面で操作しますか？（管理画面 / DB直接 / 自動判定）

## C. サブスクプラン3段階の階層と権限
1. プラン3段階は仮称 Basic / Standard / Premium で進めて良いですか？ 正式名称があれば指定してください
2. 上位プランは下位プラン視聴範囲を完全包含しますか？（Premium ⊇ Standard ⊇ Basic）それとも横並びで一部内容が異なるプラン構成ですか？
3. 各プランで視聴可能なコンテンツ／コースをどう紐付けますか？案：(a) コンテンツ／コース側に「最低プランレベル」属性、(b) プラン×コンテンツ／コースの中間テーブル、(c) コース単位でプラン紐付け
4. プラン料金は月額のみですか、年額もありますか？（Stripe の price_id を複数持つかに直結）
5. 同一プラン内での価格変更（値上げ等）は既存契約者には旧価格据え置きですか？（Stripe 側の挙動とDB上の履歴保持要否に影響）

## D. コース（Udemy 風）の構造

## E. コンテンツ・コース・プランの所属関係（最重要）
1. コンテンツ視聴の権限判定はどの粒度で行いますか？ 案：(a) ユーザー → 加入プラン → そのプランで許可されたコース → コース内のコンテンツを視聴可、(b) ユーザー → 加入プラン → プランで許可されたコンテンツ／コースを横断視聴、(c) ユーザー → 個別コース購入 → そのコース内のコンテンツのみ視聴可、(d) 上記の組み合わせ

3. サブスク中に視聴したコースを、解約後も視聴可能にしますか？（買い切り扱いに昇格）or 解約と同時に視聴不可？
4. プランによって「先行公開コンテンツ」のような時間差リリースはありますか？
