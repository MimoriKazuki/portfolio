## Stripe Webhook 処理シナリオ

1. Webhook で受信して**実処理を行う**イベントを以下から確定したい：
   - `checkout.session.completed`（サブスク／単品共通の購入完了）
   - `customer.subscription.created` / `updated` / `deleted`
   - `customer.subscription.trial_will_end`（トライアル終了予告）
   - `invoice.payment_succeeded`（継続課金成功）
   - `invoice.payment_failed`（継続課金失敗）
   - `charge.refunded`（返金）
   - `customer.updated`（メールアドレス変更等）
2. Webhook の冪等性確保はどう行うか？Stripe `event.id` を一意キーに「処理済みイベント」テーブルへ INSERT して二重処理を防ぐ方針で良いか？
3. Webhook 受信失敗時のフォールバック：
   - Stripe 側のリトライ（最大72時間）にどこまで頼るか
   - Slack 通知や管理画面の「失敗イベント一覧」画面は必要か
   - 手動再処理 API は管理者画面に置くか
4. Webhook 署名検証失敗時の挙動（ログ記録レベル・アラート要否）。
5. `checkout.session.completed` で取得する `metadata` に最低限載せる項目を確定したい（例：`user_id` / `purchase_type=subscription|course|single` / `target_id`）。
