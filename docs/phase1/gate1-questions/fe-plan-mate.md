# fe-plan-mate Gate 1 質問リスト

本リストは Phase 1 Gate 1（業務分析）段階で、画面・UX 設計のためにディレクター（Kosuke）に確認したい項目をまとめたもの。
スコープ：Eラーニング機能の大幅刷新（LP 追加 / プラン3段階化 / 買い切り / Udemy 風コース / 管理画面）。

---

## 1. LP（/e-learning 入口）の構成・必須セクション

1. LP に必須として載せたいセクションは何か（候補：ヒーロー / バリュー訴求 / プラン比較 / コース紹介 / 受講生の声 / 講師紹介 / FAQ / 実績数値 / 購入導線 / お問い合わせ）。優先度の高い順に教えてほしい。
2. ヒーローエリアの主訴求軸は何か（例：「AI 業務活用を最短で身につける」「現役 AI コンサルが講師」など）。短いコピー案・キービジュアルの素材有無も含めて確認したい。
3. 「バリュー訴求セクション」で打ち出したい価値は何か（学習効率 / 実務直結 / 講師の質 / アップデート頻度 等）。3〜5 個に絞れるか。
4. 実績数値を載せる場合、表示したい数字は何か（受講者数 / 修了者数 / 動画本数 / 法人導入社数 等）。出せる数字・出せない数字の区別。
5. 「受講生の声」を載せるか。載せる場合、現時点で出せる声・写真の素材はあるか、もしくは Phase 2 以降に追加する想定か。
6. FAQ に載せたい質問項目（候補：返金 / 解約 / 法人利用 / 視聴期限 / 修了証 / サポート対応）。優先度を教えてほしい。
7. CTA（メインアクション）は何か（「無料でログインしてサンプル視聴」「プランを見る」「LINE 登録」など）。LP 全体で何回 CTA を出すか。

## 2. LP 入口導線・未ログイン/ログイン時の挙動

1. 未ログイン状態で `/e-learning` にアクセスしたとき、LP を表示する仕様で合っているか。
2. ログイン済ユーザーが `/e-learning` にアクセスしたとき、LP を出すか、コンテンツ一覧へ自動リダイレクトするか。
3. LP からログイン導線をどこに置くか（ヘッダーのみ / ヒーロー内 CTA / 各セクションに CTA 多重配置）。
4. ログイン後の遷移先は固定で「コンテンツ一覧」か、それとも「直前に閲覧していたコンテンツ」「コース継続視聴」など状態復元するか。
5. 未ログインでもプレビュー視聴できるサンプルコンテンツを置くか（先頭数十秒のみ / 無料コンテンツのみ等）。
6. LP は SEO 対象とするか（OGP、構造化データ、メタタグ整備の優先度）。

## 3. プラン3段階の比較表 UI

1. 3 プランの名称・対象（個人ライト / 個人スタンダード / 法人 など）、価格帯（月額・年額）を確定済か。Gate 1 時点で確定していない場合、想定レンジを教えてほしい。
2. プラン比較表に並べたい比較軸は何か（視聴可能コンテンツ数 / コース受講可否 / 教材 DL 可否 / 同時視聴端末数 / 視聴期間 / サポート / 修了証 / 法人請求書）。
3. 推奨プランを強調表示するか（中央列ハイライト・「人気」ラベル等）。推奨プランはどれか。
4. 月額／年額切替トグルを置くか。年額の割引率はいくらか（例：2 ヶ月分無料）。
5. プランの「アップグレード／ダウングレード」「年額への切替」のフローを LP・マイページ上で扱うか、それともサポート問い合わせベースか。
6. キャンペーン価格・期間限定割引の表示枠を最初から確保するか。
7. 「無料体験」「初月無料」「クレカ未登録での体験」などのオファーはあるか。

## 4. 買い切り商品（コース・コンテンツ）の購入導線

1. 買い切りの対象単位は何か（個別コンテンツ単位 / コース単位 / 教材 PDF 単位 / 上記の混在）。
2. サブスクと買い切りの優先順位（同コンテンツがサブスク特典かつ買い切り対象の場合）はどう表示するか。サブスク加入中は買い切り CTA を隠すか、両方並べるか。
3. 買い切り後の視聴期限は無制限か、期間ありか。
4. 買い切りの返金ポリシー（クーリングオフ等）を画面に明示する必要があるか、利用規約リンクのみで十分か。
5. 買い切りの単価帯（例：3,000〜30,000 円）。決済画面はサブスクと共通か別か。

## 5. Udemy 風コース化・視聴画面（最重要）

1. 1 コースあたりのレッスン数の目安（例：5〜30 本）。レッスン総時間の目安（例：1〜10 時間）。
2. コース詳細ページに載せる情報項目は何か（カリキュラム一覧 / 受講対象 / 前提知識 / 学習成果 / 講師紹介 / レビュー / FAQ / プレビュー動画）。
3. 視聴画面のレイアウトは「左：レッスン一覧、メイン：動画、下：教材タブ・概要タブ・Q&A タブ」で問題ないか。スマホ時の折りたたみ方針も含めて。
4. 「次のレッスンへ自動再生」「終了後カウントダウン」など連続視聴の挙動はどうするか。
5. 章（セクション）構造を持つか、フラットなレッスン列挙か。
6. 各レッスンに紐づく教材（PDF / スライド / 演習データ）は同一画面のサイドからダウンロードできる UX で問題ないか。
7. 倍速再生・字幕・チャプターマーカーは対応するか。video プレーヤーは既存（HTML5 / Vimeo / YouTube / Cloudflare Stream 等）のどれを継続するか。
8. レッスン視聴中に Q&A・質問投稿機能を出すか（Phase 1 で含めるか、Phase 2 以降で良いか）。

## 6. 進捗 UI・修了証

1. 進捗の表示単位はレッスン単位（視聴完了フラグ）か、視聴秒数単位の％か。
2. 「視聴完了」の判定基準は何か（90% 以上視聴 / 末尾到達 / 手動でチェック）。
3. コース全体の進捗バーをどこに出すか（コース詳細 / マイページ / 視聴画面ヘッダー）。
4. 修了証の発行は Phase 1 で含めるか。含める場合、PDF 出力か Web 表示か、デザインや署名者表記の指定有無。
5. 中断箇所の自動再開（「続きから視聴」）はどの粒度で持つか（コース単位 / コンテンツ単位 / 秒単位）。
6. 受講履歴・修了履歴をユーザーマイページに見せるか。法人プランで管理者が受講者の進捗を見る要件はあるか。

## 7. 視聴権限による表示分岐・CTA

1. 未ログイン時のコンテンツカードの表示はどうするか（鍵アイコン / ぼかし / 「ログインで視聴」CTA）。
2. ログイン済かつ未加入時、コンテンツカードに「このプランで視聴可」「買い切りで視聴可」をどう表示するか。
3. 視聴画面に直接アクセスしたが権限がない場合、コース詳細へ戻すか、プラン選択モーダルを出すか。
4. プラン加入導線（アップセル CTA）はどの画面に何回出すか（ホーム / コンテンツ一覧 / 視聴画面 / マイページ）。過度な煽りを避ける方針か。
5. 「無料プレビュー」のラベルやコンテンツの存在を明示するか。

## 8. 既存購入者・弊社メンバーの表示

1. 既存購入者 6 名は「弊社メンバー」扱いで全機能視聴可。マイページ上で「弊社メンバーステータス」を可視化するか、それとも内部ステータスとしてユーザーには見せないか。
2. 「弊社メンバー」のラベルを画面上に出す場合、何と表記するか（社員 / メンバー / 関係者 / その他）。プラン表記を「メンバー（特別）」のように出すか。
3. 弊社メンバーがコンテンツ一覧を見たとき、購入 CTA・プラン CTA は非表示にするか、薄く出すか。

## 9. 管理画面（コース・プラン・買い切り・購入履歴）

1. コース作成画面のレイアウトはどう想定するか（基本情報フォーム ＋ レッスン並び替えエリア＋公開設定）。レッスンの並び替えはドラッグ＆ドロップ前提か。
2. コース ↔ コンテンツの紐付け UI は「コース側からコンテンツを選択して追加」型か、「コンテンツ側からコースに追加」型か、両方か。
3. プラン編集画面で操作する項目（プラン名 / 価格 / 視聴可能カテゴリ / 視聴可能コース / Stripe Price ID / 公開フラグ / 推奨フラグ）。Stripe Price ID は画面で扱うか、env で固定するか。
4. 買い切り商品の管理画面は新規追加か、既存 `/admin/e-learning` 内に統合するか。商品単位の在庫・販売停止フラグはあるか。
5. 購入履歴一覧で必要な絞り込み軸（プラン / 商品 / ユーザー / 期間 / ステータス）。CSV エクスポートは Phase 1 で必要か。
6. 受講者進捗の管理画面ビューは Phase 1 で必要か（コースごとの完了率一覧など）。
7. プラン解約・返金処理は管理画面で操作するか、Stripe ダッシュボード側に寄せるか。
8. コンテンツの公開／非公開／予約公開、改訂版差し替え UI の要否。

## 10. レスポンシブ・デバイス対応

1. 想定する利用デバイス比率（PC / タブレット / スマホ）。スマホ視聴を主要シナリオに含めるか。
2. スマホ時の視聴画面レイアウト（レッスン一覧をボトムシートで開閉 / タブ切替 / 別画面遷移）。
3. LP のブレークポイント方針（モバイルファーストか PC ファーストか、対応最小幅）。
4. 動画再生中の縦持ち／横持ち切替対応（フルスクリーン誘導、再生中の UI 表示時間）。
5. 管理画面はスマホ対応するか、PC 前提で良いか。

## 11. デザイントークン・既存デザインシステム

1. カラーパレット（プライマリ / セカンダリ / アクセント / セマンティック）の既定義はあるか。Tailwind の設定で固まっているか、デザイン資料（Figma 等）に正があるか。
2. タイポグラフィ（フォントファミリー、見出しスケール、本文サイズ、行間）の既定義はあるか。和文・欧文のフォント指定有無。
3. スペーシング（4px / 8px ベース）と角丸・シャドウのトークン定義はあるか。
4. ブランドの「らしさ」を保つために守るべきルール（例：ボタンは塗り＋シャドウなし、見出しはゴシック太字）はあるか。
5. ロゴ・アイコンライブラリ（lucide / heroicons 等）の指定。
6. ダークモードの対応要否。
7. アクセシビリティ要件（WCAG 準拠レベル、コントラスト比、キーボード操作、スクリーンリーダー対応）。
8. Figma など外部デザイン資料は存在するか。存在する場合、URL を共有してほしい。

## 12. その他の運用・コンテンツ周り

1. お知らせ・新着コンテンツの導線（LP 内 / マイページ / 通知バッジ）。
2. メール通知（購入完了 / プラン更新 / 新着コース）と画面の役割分担。Phase 1 でメールテンプレ画面が必要か。
3. SEO・OGP の重要度（プラン LP・コース詳細のシェア時 OGP 画像、メタ説明文）。
4. 法的表記（特定商取引法 / 利用規約 / プライバシーポリシー）はチェックアウト前にどこで提示するか。既存ページを流用するか新設するか。
5. クーポン・割引コードの UI を Phase 1 で扱うか。
6. 多言語対応（日本語のみで進めるか、将来英語対応の余地を残すか）。
