# 業務分析（Gate 1）

## 概要

AI駆動研究所 ポートフォリオ／メディアサイトに搭載されているEラーニング機能を、Phase 1 で「コース＋単体動画の買い切り販売」モデルへ刷新する。
未ログインユーザー向けにランディングページ（LP）で買い切り訴求を行い、ログイン済みユーザーに対しては「無料／有料」「コース／単体動画」の4パターンのコンテンツを提供する。サブスクリプションは採用せず、購入したコース・単体動画は永続視聴可能（Udemy 同様）とする。

対象は B to C 個人向けで、法人向け契約（e_learning_corporate_*）は Phase 1 スコープ外。

## 業務フロー

ECサイト＋動画視聴サービスの典型的なフローに準じる。主要フェーズを時系列で整理する。

| フェーズ | 主体（誰が） | タイミング | 入力 | 出力 | 備考 |
|---------|------------|-----------|------|------|------|
| 1. 商品（コース・単体動画）の整備 | 管理者 | 随時 | コース情報・章構成・動画ファイル・価格 | コース／単体動画レコード（公開／非公開状態） | Stripe Price は1コース／1動画につき1つ作成 |
| 2. LP 流入・サービス紹介 | 一般訪問者（未ログイン） | LP 公開後 | LP の閲覧 | ログイン／会員登録動線 | ログイン必須（Udemy 同様） |
| 3. 会員登録／ログイン | 訪問者 | 購入前または視聴前 | Google OAuth 認証 | `e_learning_users` レコード（109件は既存継続） | 既存購入者6名は `has_full_access=true` 自動付与 |
| 4. コンテンツ閲覧（無料分） | ログインユーザー | ログイン後 | 視聴操作 | 視聴ログ・進捗 | 無料コース／無料単体動画／有料コース内 is_free 動画 |
| 5. 購入決済（コース／単体動画） | ログインユーザー | 有料コンテンツの購入ボタン押下時 | Stripe Checkout（mode: payment） | `e_learning_purchases` レコード | One-time payment、永続視聴権 |
| 6. 購入完了 Webhook 受信 | システム | 決済成功時 | `checkout.session.completed` | 購入レコード作成・視聴権限付与 | Stripe → Webhook → DB |
| 7. コンテンツ視聴 | 購入済ユーザー | 購入後いつでも | 視聴操作 | 視聴進捗データ | 永続視聴可・期限なし |
| 8. ブックマーク登録 | ログインユーザー | 任意 | お気に入り登録操作 | `e_learning_bookmarks` レコード | 既存3件継続 |
| 9. 返金処理 | 管理者（Stripe ダッシュボード側） | 顧客対応時 | Stripe 上の返金操作 | `charge.refunded` Webhook → 視聴権限剥奪 | 返金ポリシー・運用は次点質問で確定予定 |
| 10. 運用メンバーへの全アクセス権付与 | 管理者 | 随時 | 管理画面操作 | `has_full_access=true` フラグ更新 | 手動切替（自動判定ロジックは持たない） |

業務フロー図：
```
[未ログイン]
    │
    ▼
[LP（/e-learning）] ── 買い切り訴求
    │
    ▼
[ログイン（Google OAuth）] ── ログイン必須
    │
    ├──→ [無料コンテンツ視聴]（無料コース／無料単体動画／is_free動画）
    │
    └──→ [有料コンテンツ詳細]
              │
              ▼
         [Stripe Checkout（mode: payment）]
              │
              ▼
         [購入完了 Webhook（checkout.session.completed）]
              │
              ▼
         [視聴権限付与・永続視聴可]

[管理者]
    │
    ├──→ [コース管理・動画管理・価格設定]
    ├──→ [運用メンバーへ has_full_access 付与]
    └──→ [購入履歴閲覧]（返金は Stripe ダッシュボード側で実施）
              │
              ▼
         [charge.refunded Webhook → 権限剥奪]
```

## 業務制約・ルール

### 視聴権限の優先順位（最重要）

確定事項より、以下の順で視聴可否を判定する。

```
① has_full_access = true            → 全動画視聴可（無条件）
② コース購入済み                       → そのコース内の全動画視聴可
③ 単体動画購入済み                     → その動画視聴可
④ 動画が is_free フラグ true          → ログインユーザー全員視聴可
⑤ それ以外                            → 視聴不可
```

- ログイン必須：未ログインユーザーは LP のみ閲覧可（コンテンツ視聴一切不可）
- 購入したコース・単体動画は **永続視聴可**（期限・解約・失効の概念なし）
- 返金された場合（`charge.refunded` Webhook 受信時）は権限剥奪

### コンテンツ販売の制約

- **コース内動画は個別購入できない**（コース単位購入のみ）。「このコースの第3話だけ買う」は不可
- **単体動画はコースに所属しない**。単体販売のみ
- **既存15動画はすべて単体動画として継続**。新規作成するコースには既存動画を含めない
- 同一動画ファイルを「コース内動画」と「単体動画」の両方として販売することはしない

### コース構造の制約

| 項目 | 設定 |
|------|------|
| コース：動画の関係 | **1対多**（1コース＝複数動画、1動画は1コース固定） |
| 動画順序 | **あり**（章内の動画は順序を持つ） |
| 章構造 | **あり**（コース → 章 → 動画 の3階層） |
| 視聴ロック | **なし**（順序を持つが先頭から見なくてもよい・自由視聴） |
| コース内動画の無料／有料 | **`is_free` フラグで個別マーキング可**（有料コース内に無料動画を混在させられる） |

### has_full_access フラグの運用ルール

- カラム名：`has_full_access`（bool）
- 配置：`e_learning_users` テーブル
- デフォルト：`false`
- 既存購入者6名：移行時に自動で `true` に設定
- 運用メンバー：管理画面で **手動切替**（自動判定ロジックは Phase 1 では実装しない）
- ステータス名：「フルアクセス」を採用（一般ユーザーはこのフラグなし）

### 支払い・返金の制約

- 採用モデル：**買い切りのみ**（サブスクは廃止）
- 決済方法：Stripe Checkout の `mode: 'payment'`（One-time payment）
- 各コース・各単体動画につき **One-time Price を1つずつ** 作成
- 既存の `STRIPE_E_LEARNING_PRICE_ID` 固定方式は廃止（→ DB 側でコース／動画ごとに Stripe Price ID を保持）
- 必要な Webhook イベント：
  - `checkout.session.completed`（購入完了 → 権限付与）
  - `charge.refunded`（返金 → 権限剥奪）
- サブスク系イベント（`customer.subscription.*` / `invoice.*`）は **不要**

### 履歴保持要件

- 購入履歴（`e_learning_purchases`）：永続保持（税務観点・領収書再発行対応）
- 既存購入レコード6件：物理削除せず保持。該当6名へ `has_full_access=true` を自動付与
- 退会時のデータ保持ポリシー：次点質問Q19で確定予定（暫定：購入履歴は保持・個人情報マスキング想定）

### 管理者ロールの制約

- **現状維持**：「auth.users に存在＝管理者」のみ。Phase 1 でロール分離は行わない
- 監査ログ（誰がいつ何を変更したか）：Phase 1 では実装しない
- 管理者全員が全管理操作可（プラン編集・コース作成・購入履歴閲覧・運用メンバー化）

### 設計負債6項目の Phase 1 対応（一括承認済）

| # | 内容 | Phase 1 対応 |
|---|------|------------|
| 1 | `e_learning_bookmarks.user_id` が `auth.users.id` 参照／`e_learning_purchases.user_id` が `e_learning_users.id` 参照 → 不統一 | **`e_learning_users.id` 参照に統一する** |
| 2 | `e_learning_purchases.content_id` NULL 許容（全コンテンツ買い切り = NULL） | **新スキーマで「コース購入／単体動画購入」を区別する設計に再整理** |
| 3 | `e_learning_users` に `role` カラムなし | **`has_full_access` フラグ追加で対応（ロール分離は行わない）** |
| 4 | `e_learning_purchases` に `stripe_payment_intent_id` なし | **Phase 1 でカラム追加**（返金照合のため） |
| 5 | 管理者判定が「ログイン＝admin」のみ | **現状維持**（ロール分離なし） |
| 6 | `ADMIN_EMAIL` 環境変数がコード未参照 | **Phase 1 で削除**（混乱防止） |

### バックデート制限・締日

- 該当なし（本業務は日次／月次の業務処理を持たない）

## 既存システムからの移行

本プロジェクトは **既存稼働中の Supabase 環境を継続使用** する。新規構築ではなく、既存スキーマに追加する形で Phase 1 を進める。

### 既存資産の移行方針

| 既存資産 | 件数 | 移行方針 |
|---------|-----|---------|
| `e_learning_users` | 109 件 | そのまま継続 |
| `e_learning_bookmarks` | 3 件 | そのまま継続（ただし FK 参照先を `e_learning_users.id` に統一：設計負債1の対応） |
| `e_learning_contents` | 15 件 | **単体動画として継続**（コースには入れない） |
| `e_learning_categories` | 6 件 | そのまま継続（コース化に伴う再編なし） |
| `e_learning_materials` | 7 件 | そのまま継続 |
| `e_learning_purchases` | 6 件 | レコードは保持。該当6名のユーザーに `has_full_access=true` を自動付与 |
| `e_learning_corporate_customers` | 0 件 | スコープ外（将来用に受け皿のみ存続） |
| `e_learning_corporate_users` | 0 件 | スコープ外（将来用に受け皿のみ存続） |

### 既存スキーマのコード化（Phase 1 内で実施）

- 担当：plan-lead
- 方法：supabase MCP の読み取り専用クエリ（`list_tables` / `list_migrations` / `get_advisors` / `execute_sql` で SELECT のみ）で現状スキーマを取得し、`supabase/migrations/0000_initial_existing_schema.sql` として保存する
- 経緯：開発初期はマイグレーションファイルで運用していたが、途中で supabase MCP 経由のダッシュボード編集に切り替えた結果 DDL がリポジトリに残らない状態になった。Phase 1 でこれを正常化する

### 並行稼働期間

なし。Phase 2/3 で新スキーマを適用し、リリース時点で旧仕様（サブスク準備・全コンテンツ買い切り）は廃止する。
ただし既存購入者6名・既存ユーザー109名・既存ブックマーク3件・既存単体動画15件は **すべて継続利用** する。
